Que.1 Are The HTML Tags And Elements The Same Thing?

Ans-

Tags: These are the markup characters used to define the structure of an HTML document. They come in pairs: an opening tag and a closing tag. Forinstance, '< >' is the opening tag for a paragraph, and '< />' is the closing tag. Tags define where an element starts and ends.

Elements: An element consists of an opening tag, content, and a closing tag. It's the complete set that represents a specific component or structure within an HTML document. For example, the '< >' tag, its content, and the '' closing tag together form a paragraph element.

Que.9 How is Cell Padding different from Cell Spacing?

Ans- Cell Padding: Cell padding refers to the space between the content of a table cell and its border. When you set cell padding, you're defining the amount of space that exists between the content of a cell and the cell's border. This padding applies to each individual cell in the table.
Cell Spacing: Cell spacing, on the other hand, refers to the space between the cells in a table. It defines the amount of space or gap between adjacent cells in the table.
In modern web development, using attributes like cellpadding and cellspacing directly on the table tag is considered somewhat outdated. Instead, CSS is preferred for defining spacing, padding, and other styles.

Que.11 What is the difference between a block-level element and an inline element?

Ans- elements and inline elements are two fundamental display types for HTML elements, and they differ in how they are rendered and how they interact with other elements.
1.Block-level elements: Typically start on a new line and take up the full width available (unless specified otherwise by CSS). Examples of block-level elements include 'div', 'p', 'h1' to 'h6', 'ul', 'li', 'table', etc.
You can set width, height, margins, padding, and borders for block-level elements.
2.Inline elements:Do not start on a new line and only take up as much width as necessary. Examples of inline elements include 'span', 'a', 'strong', 'em', 'img', 'input', etc. You cannot set width, height, or margins for inline elements

Que.13 What is the use of an iframe tag?

Ans- 'iframe' tag in HTML stands for "inline frame." It's used to embed another document or webpage within the current HTML document. Essentially, it creates a window through which an external webpage or content can be displayed within the current page.
Here are some common uses of the 'iframe' tag:
1.Embedding external content: You can embed content from another website or web page within your own. For example, displaying a map from Google Maps or embedding a YouTube video.
2.Embedding local content: You can display content from within the same website or server by specifying the relative or absolute path to the file.
3.Displaying external content securely: 'iframe' elements can help contain potentially unsafe or untrusted content from external sources within a sandboxed environment, preventing it from affecting the main page's functionality.
4.Creating inline frames: In some cases, 'iframe' can be used for dividing a webpage into sections, allowing separate browsing contexts within the same document.

Que.16 How are active links different from normal links?

Ans- Active links, also known as "active state" or "active link state," differ from normal links (or default links) based on their appearance or behavior when a user interacts with them.

Normal links ('a' tags in HTML) have various states:

Normal/Default State: is the initial appearance of a link when it's not interacted with.
Hover State: This is when a user hovers their cursor over the link.
Visited State: Links that have been visited by the user, usually styled differently from normal links to indicate they've been accessed before.
Active State: This is the state when a link is being activated or clicked by the user.
The active state occurs while a link is being clicked or activated by the user. It's a transient state that's typically seen when the mouse button is pressed down on the link. The active state usually provides immediate feedback to the user, indicating that their click has been registered.

Que.18 What is SVG?

Ans-SVG stands for Scalable Vector Graphics. It's an XML-based vector image format for 2D graphics that supports interactivity and animation. Unlike raster image formats like JPEG or PNG, which use a grid of pixels to represent images, SVG uses XML to describe shapes, curves, text, and other graphical elements.
Key features of SVG include:
Scalability: images can be scaled up or down without losing quality. They remain sharp and clear at any size because they are based on mathematical descriptions of the shapes.
Editable: Since SVG is based on XML, SVG images can be created and edited with text editors or specialized vector graphics software like Adobe Illustrator, Inkscape, or Sketch.
Compact File Size: SVG files tend to be smaller in size compared to raster image formats, especially for images with simple shapes and limited colors.
Support for Interactivity and Animation: SVG supports scripting and animation through JavaScript, allowing for interactive elements, transitions, and animations within the image itself.

Que.19 What is difference between HTML and XHTML?

Ans-HTML (HyperText Markup Language) and XHTML (Extensible HyperText Markup Language) are both markup languages used to create web pages, but they have some differences in syntax and rules.
1.Syntax:
HTML: has a more forgiving syntax. It allows for some mistakes, such as unclosed tags, omitted quotes around attribute values, and mixed case for tags and attributes.
XHTML:follows stricter syntax rules derived from XML. It requires properly nested tags, lowercase tag names, attribute values enclosed in quotes, and self-closing tags for empty elements.
2.Parsing:
HTML: HTML parsers are generally more lenient and can handle some errors, allowing the browser to render the content even if there are minor issues.
XHTML: XHTML parsers are stricter and will often fail to render the page if it encounters errors in the markup. It requires well-formed XML syntax to function correctly.
3.MIME Type:
HTML: HTML documents are served with the MIME type 'text/html'.
XHTML: XHTML documents are served with the MIME type 'application/xhtml+xml'.
4.Compatibility:
HTML: HTML is more widely supported by browsers and has better backward compatibility with older web pages.
XHTML: While XHTML was introduced as the next step in HTML development, its stricter rules and syntax caused some issues with existing web pages, leading to slower adoption.
5.Extensibility:
XHTML: Being based on XML, XHTML is more extensible and can be integrated with other XML-based languages and technologies.
HTML: Traditional HTML has limitations in terms of extensibility due to its syntax and parsing rules.

Que.20 What are logical and physical tags in HTML?

Ans-In HTML, there's a distinction between logical tags and physical tags that was more relevant in older versions of HTML and hasn't been as prominent in modern HTML5.
1.Physical Tags:
Physical tags refer to tags that describe the appearance or formatting of content rather than its meaning or structure. Examples of physical tags include 'b' (bold), 'i' (italic), 'font' (font formatting), and 'strike' (strikethrough). These tags directly manipulate the visual presentation of text but don't convey the semantic meaning of the content.
2.Logical Tags:
Logical tags, on the other hand, are focused on defining the structure and meaning of content rather than its appearance. They convey the semantics of the content. Examples include 'header', 'nav', 'section', 'article', 'footer', etc., introduced in HTML5. These tags provide a clearer and more meaningful structure to web documents, aiding accessibility,search engine optimization,and understanding of the content's purpose and hierarchy.
